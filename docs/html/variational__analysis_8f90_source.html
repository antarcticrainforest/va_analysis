3 !
4 ! This is the main variational analysis program. It has been directly ported from the IDL procedure called assim_twpice.f90, with
5 ! apppropriate Fortran 95 modifications. The IDL procedure assumed the user had already pre-processed the input data using a suite
6 ! of IDL scripts. The results of this pre-processing were assumed to be already loaded into memory once the main variational
7 ! analysis procedure was started. We can't do this with Fortran 95 (unless we make one huge monolithic program which does both
8 ! the pre-processing and variational analysis), so instead we read the preprocessed data from a NetCDF file.
9 !
10 ! Inputs: * Pre-processed data in NetCDF format. These can be generated using the other software distributed with this code:
11 ! see the documentation.
12 ! * Settings specific to the variational analysis are set in the SETTINGS module (settings.f90). If you change any of
13 ! these settings, you will need to re-compile the software.
14 !
15 ! Tim Hume.
16 ! 24 August 2006.
17 !***********************************************************************************************************************************
