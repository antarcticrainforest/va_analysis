1 3D Variational Data Assimilation of ECMWF Analysis Model Data {#mainpage}
2 =============================================================
3 This repository contains the source-code of a 3D Variational Analysis (3D Var)
4 
5 For more details, see the scientific publications:
6 
7 * [Constrained Variational Analysis of Sounding Data Based on Column-Integrated Budgets of Mass, Heat, Moisture, and Momentum: Approach and Application to ARM Measurements](https://doi.org/10.1175/1520-0469(1997)054<1503:CVAOSD>2.0.CO;2)
8 
9 * [Observed Large-Scale Structures and Diabatic Heating and Drying Profiles
10 during TWP-ICE](http://doi.org/10.1175/2009jcli3071.1)
11 
12 The repository is available on GitHub: [github.com/antarcticrainforest/va_analysis](https://github.com/antarcticrainforest/va_analysis)
13 and includes the following directories:
14 
15  * 2D_create - directory with scripts to create the 2D input fields for the 3D Var
16  * 3D_create - and the scripts for the 3D input field
17  * process_MWR - scripts to process the microwave based observations
18  * process_rain - scripts to process the radar based rainfall estimates
19  * src - the source code for the variational analysis
20  * docs - documentation for GitHub-Pages
21

64 Data and format
65 ---------------
66 You might need to [create](https://www.archive.arm.gov/armuserreg/#/new>) an
67 account at the Atmospheric Radiation Measurement (ARM) Climate Research Programm.
68 Once you have an account you will need to download the following data:
69 
70 Suppose you want to run the analysis for Feb 2014:
71 Go to the [arm data catalogue](http://www.archive.arm.gov/discovery/#v/results/s) and
72 search for :
73 
74 * Surface data from ecmwf model (e.g. twpecmwfsurfX1 for data over Darwin)
75 * Supplement data from ecmwf model (e.g. twpecmwfsuppX1 for data over Darwin)
76 * Upper atmosphere data from ecmwf model (e.g. twpecmwfupaX1 for data over Darwin)
77 * Microwave Water Radiometer measurements (e.g. twpmwrlosC3.b1 for data from Darwin)
78 * Surface measurements (e.g. twpmetC3.b1 for data from Darwin)
79 
80 If you want to retreive data from Darwin than just copy and paste the acronyms.
81 If not, you can choose any facility that is offered on the ARM site. You only have
82 to make sure to download the supplemental model data from ecmwf (*ecmwfsupp*), the
83 surface data (*surf*), upper atmosphere data (*upp*) from ecmwf, microwave
84 radiometer (*mwrlos*) and surface met measurements (*met*). The prefix twp stands
85 for tropical western pacific and the suffix X1/C3.b1 for model data/observations
86 over Darwin. Replace the prefixes and suffixes by the desired region and station.
87 The acronyms can be found on the search page under the sites and facility bars.
88 
89 Once you downloaded the data make sure all files are located in ``one`` folder.
90 
91 You also have to download rainfall radar data from your preferred side. Make
92 sure that the files come in daily format. The data can either be in ascii or netcdf format. Once again make
93 sure that all files are stored in the same folder.

96 Building
97 ========
98 The ``setup.py`` script in the ``src`` folder helps you creating a Makefile
99 for building the 3D Var program. Some important environment variables are:
100 
101 * FC Fortran compiler command
102  [default gfortran]
103 * CC C compiler command
104  [default gcc]
105 * FCFLAGS Fortran compiler flags
106  [default -O3 -Wpedantic -fimplicit-none -fsign-zero]
107 * CFLAGS C compiler flags
108  [default -O3 -Wpedantic]
109 * LD_LIBRAY_PATH linker flags, e.g. -L<lib dir> if you have libraries in a
110  nonstandard directory <lib dir>
111  [default %s]
112 * INCLUDE include flags e.g. -I<include dir> in a
113  the headers in a nonstandard directory <include dir>
114  [default %s]
115 * FLIBS fortran libraries to pass to the linker, e.g. -l<library>
116  [default netcdff]
117 * CLIBS c libraries to pass to the linker, e.g. -l<library>
118  [default netcdf,m]
119 
120 Use these variables to override the choices made by setup.py or to help
121 it to find libraries and programs with nonstandard names/locations.
122 
123 To create the Makefile simply type::
124 ```bash
125 $: python setup.py
126 ```
127 Comiple the source code with::
128 ```bash
129 $: make && make install
130 ```

139 Usage
140 =====
141 Once everything is downloaded and working ok run the ``preprocess.sh`` script.
142 The script takes the following command line arguments:
143 
144 * -a , --arminput : Input dir of the atmospheric data
145 * -r , --raininput : Input dir of the radar data
146 * -o , --output : Output dir of the varational analysis
147 * -v , --va_input : Input dir of the variational analysis
148 
149 Submission to a Computing Cluster
150 ---------------------------------
151 To speed computation up and and run multiple seasons paralell an option to submit all
152 (pre-)processing to a Linux computing has been implemented.
153 To create a submission script that submits the (pre-)processing scripts to a cluster simply run:
154 ```bash
155 $: python setup.py --BATCH=system
156 ```
157 
158 Where ```system``` can either be pbs or slurm. You will be asked for a email address and a project ID for resource
159 allocation (optional). This will create a ``submit`` script in the main va_analysis directory. You can change the
160 resource allocation if necessary. The output of the (pre-)processing will be redirected into ``Jobs`` in the va_analysis main directory.
161 
162 For more options and how to use this script see:
163 ```bash
164 $: submit_system.sh --help
165 ```
